## 摘要
在许多机器人操纵任务中，机器人需要重复解决运动规划问题，这些问题的主要区别在于目标物体及其相关障碍物的位置，而周围的工作空间保持不变。先前的研究表明，利用经验和离线计算可以加速重复的规划查询，但它们缺乏覆盖连续任务空间的保证，且需要存储庞大的解库。在这项工作中，我们提出了 COAD，这是一个在连续目标参数化任务空间上提供常数时间规划的框架。COAD 将连续任务空间离散化为有限个任务覆盖区域。它并不离线规划并存储每个区域的解，而是通过仅解决代表性的根问题来构建压缩库。其他问题则通过对这些根解进行快速自适应来处理。在查询时，系统以常数时间检索根运动，并使用轻量级自适应模块（如线性插值、动态运动基元或简单的轨迹优化）将其调整至所需目标。我们在仿真和真实世界的各种机械臂及环境中评估了该框架，结果表明 COAD 在保持高成功率和亚毫秒级查询的同时，实现了运动库的大幅压缩，在效率和路径质量上均优于基准方法。源代码可在 https://github.com/elpis-lab/CoAd 获取。

## Abstract
In many robotic manipulation tasks, the robot repeatedly solves motion-planning problems that differ mainly in the location of the goal object and its associated obstacle, while the surrounding workspace remains fixed. Prior works have shown that leveraging experience and offline computation can accelerate repeated planning queries, but they lack guarantees of covering the continuous task space and require storing large libraries of solutions. In this work, we present COAD, a framework that provides constant-time planning over a continuous goal-parameterized task space. COAD discretizes the continuous task space into finitely many Task Coverage Regions. Instead of planning and storing solutions for every region offline, it constructs a compressed library by only solving representative root problems. Other problems are handled through fast adaptation from these root solutions. At query time, the system retrieves a root motion in constant time and adapts it to the desired goal using lightweight adaptation modules such as linear interpolation, Dynamic Movement Primitives, or simple trajectory optimization. We evaluate the framework on various manipulators and environments in simulation and the real world, showing that COAD achieves substantial compression of the motion library while maintaining high success rates and sub-millisecond-level queries, outperforming baseline methods in both efficiency and path quality. The source code is available at https://github.com/elpis-lab/CoAd.

在工业生产（如分拣、包装）中，机器人常需在固定工作空间内重复执行任务，唯一的变量是目标物体的位姿（在连续空间内变化）。
*   **核心挑战**：传统的采样算法（如 RRT）缺乏运行时间保证；基于经验的库方法虽然快，但难以覆盖连续的任务空间，且随着覆盖率提高，存储开销呈爆炸式增长。
*   **研究目标**：开发一个名为 **COAD** 的框架，旨在实现**常数时间（Constant-Time）**的在线规划，同时通过**压缩库**技术显著降低存储需求，并保证对连续目标空间的覆盖。

#### 2. 论文提出的方法论
COAD 的核心思想是将连续的任务空间离散化为有限的“覆盖区域”，并利用路径的几何相似性进行压缩。
*   **任务覆盖区域 (TCR)**：论文引入了 TCR 概念。不同于传统的 TSR（定义目标位姿集合），TCR 定义了“一个固定的末端执行器位姿能够解决的所有物体位姿集合”。通过数学证明，作者展示了可以用有限数量的 TCR 单元覆盖整个连续任务空间。
*   **离线库构建与压缩**：
    1.  **离散化**：将任务空间划分为轴对齐的网格单元。
    2.  **根运动（Root Motion）生成**：为某些单元规划一条“根路径”。
    3.  **自适应扩展**：尝试将该根路径通过轻量级算法适配到邻近单元。如果适配后的路径无碰撞且可行，则邻近单元只需存储指向该根路径的索引，从而实现高达 97% 的压缩率。
*   **在线查询与自适应**：
    *   查询时，通过哈希表在 $O(1)$ 时间内检索对应的根运动。
    *   使用三种可选的自适应模块进行微调：**线性插值 (LI)**、**动态运动基元 (DMP)** 或**简单轨迹优化 (STO)**。

#### 6. 论文的主要结论与发现
*   **极高的压缩率**：在保持 100% 成功率的前提下，COAD 实现了 **72% 至 97%** 的存储压缩。
*   **亚毫秒级查询**：在线查询时间低至 **0.03ms**（LI 模式），比 RRT-Connect 快 2-3 个数量级。
*   **高质量路径**：相比于 RRT 产生的随机路径，COAD 适配出的路径在长度和顺滑度上表现更好，尤其是使用 DMP 适配时。
*   **连续空间覆盖**：证明了通过 TCR 离散化可以确保连续任务空间内的任何查询都能找到对应的预存解。

#### 7. 优点
*   **理论支撑**：通过 TCR 的数学定义解决了连续空间覆盖的完备性问题。
*   **极速响应**：常数时间的检索机制非常适合对实时性要求极高的工业场景。
*   **灵活性**：框架支持多种自适应模块，用户可以根据算力资源选择最简单的插值或更复杂的优化。
*   **实战价值**：大幅降低了经验库的内存占用，使得在嵌入式控制器上部署大规模规划库成为可能。

#### 8. 不足与局限
*   **环境静态假设**：目前主要针对“半静态”环境（目标物变，背景不变）。如果工作空间内的障碍物发生移动，预存的库可能失效。
*   **根节点选择策略**：目前的根区域选择是随机的，可能不是最优的压缩方案。
*   **离线计算量**：虽然在线很快，但离线构建 TCR 覆盖和验证适配性需要大量的预计算时间。
*   **应用限制**：主要适用于目标位姿在特定 SE(3) 范围内波动的任务，对于完全随机、无规律的任务空间效果有限。
