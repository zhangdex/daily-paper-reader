## 摘要
虽然大语言模型（LLMs）为机器人任务规划提供了语义灵活性，但它们易受幻觉和逻辑不一致性的影响，限制了其在长程领域中的可靠性。为了弥合非结构化环境与严谨规划合成之间的差距，我们提出了 DUPLEX，这是一种代理式双系统神经符号架构，它将 LLM 严格限制在模式引导的信息提取任务中，而非端到端规划或代码生成。在我们的框架中，前馈快系统（Fast System）利用轻量级 LLM 从自然语言中提取实体、关系等，并将其确定性地映射为经典符号规划器的规划领域定义语言（PDDL）问题文件。为了解决复杂或描述不足的场景，慢系统（Slow System）仅在规划失败时激活，利用求解器的诊断信息驱动高容量 LLM 进行迭代反思和修复。在 12 个经典和家庭规划领域进行的广泛评估表明，DUPLEX 在成功率和可靠性方面均显著优于现有的端到端和混合 LLM 基准。这些结果证实，关键不在于让 LLM 更好地进行规划，而是将其限制在它所擅长的部分——结构化语义落地（semantic grounding），并将逻辑规划合成交给符号规划器。

## Abstract
While Large Language Models (LLMs) provide semantic flexibility for robotic task planning, their susceptibility to hallucination and logical inconsistency limits their reliability in long-horizon domains. To bridge the gap between unstructured environments and rigorous plan synthesis, we propose DUPLEX, an agentic dual-system neuro-symbolic architecture that strictly confines the LLM to schema-guided information extraction rather than end-to-end planning or code generation. In our framework, a feed-forward Fast System utilizes a lightweight LLM to extract entities, relations etc. from natural language, deterministically mapping them into a Planning Domain Definition Language (PDDL) problem file for a classical symbolic planner. To resolve complex or underspecified scenarios, a Slow System is activated exclusively upon planning failure, leveraging solver diagnostics to drive a high-capacity LLM in iterative reflection and repair. Extensive evaluations across 12 classical and household planning domains demonstrate that DUPLEX significantly outperforms existing end-to-end and hybrid LLM baselines in both success rate and reliability. These results confirm that The key is not to make the LLM plan better, but to restrict the LLM to the part it is good at - structured semantic grounding - and leave logical plan synthesis to a symbolic planner.

*   **研究动机**：大语言模型（LLM）在处理机器人长程任务规划时，虽然具备出色的语义理解能力，但极易产生“幻觉”和逻辑不一致性。现有的“LLM作为规划器”或“LLM直接生成PDDL代码（如LLM+P）”的方法，往往因为语法错误或逻辑遗漏导致规划失败。
*   **核心目标**：弥合非结构化环境描述与严谨符号规划之间的鸿沟。论文主张：**不应让LLM直接做逻辑规划，而应将其限制在它擅长的“信息提取（IE）”任务上**，将逻辑合成交给成熟的符号规划器。

### 2. 方法论
DUPLEX 采用了一种受认知科学启发的**双系统神经符号架构**：
*   **快系统（Fast System）**：
    *   **引导式信息提取**：使用轻量级 LLM（如 Qwen3-8B）根据预定义的 Schema（模式），从环境描述中提取实体、关系和状态，输出结构化的 JSON。
    *   **确定性映射**：通过预编写的脚本将 JSON 转换为 PDDL 问题文件。这种方式避开了 LLM 生成代码时的语法脆弱性，保证 100% 的语法正确。
    *   **符号规划**：调用经典规划器（如 Fast Downward）生成动作序列。
*   **慢系统（Slow System - 代理式反思）**：
    *   **触发机制**：仅在快系统规划失败（如目标不可达）时激活。
    *   **迭代修复**：利用高容量 LLM（如 GPT-4o）作为诊断代理，分析规划器的错误信息，识别遗漏的隐式状态或语义错误，并迭代修改 PDDL 文件直至成功。
*   **分层验证**：包含规则级（语法）、语义级（词汇一致性）和逻辑级（规划器反馈）三重校验。

    *   **家庭具身领域（4个）**：如 PC Assembly（电脑组装）、Cleaning（清洁）等，侧重语义复杂度和隐式约束。
*   **基准模型（Baselines）**：
    *   **LLM-as-Planner**：GPT-4o 直接生成动作序列。
    *   **LLM+P**：GPT-4o 直接将问题翻译为 PDDL 代码。

### 6. 主要结论与发现
*   **性能飞跃**：DUPLEX 在 IPC 领域达到 97.5% 的成功率（远超 LLM+P 的 71.9%）；在家庭领域达到 83.5%（远超 LLM+P 的 20.0%）。
*   **核心发现**：将 LLM 的角色从“代码生成器”降级为“信息提取器”能显著提高系统的鲁棒性。
*   **反思的重要性**：慢系统通过规划器的反馈进行自我修正，是解决描述不足（underspecified）场景的关键。

### 7. 优点
*   **架构优雅**：结合了连接主义（LLM 的语义能力）和符号主义（规划器的逻辑严谨性），取长补短。

*   **成本效益**：日常任务由轻量级模型处理，仅在必要时调用昂贵的大模型，兼顾了效率和性能。
