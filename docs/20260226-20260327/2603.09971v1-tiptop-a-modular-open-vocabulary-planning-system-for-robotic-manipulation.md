## 摘要
我们提出了 TiPToP，这是一个可扩展的模块化系统，它将预训练的视觉基础模型与现有的任务与运动规划器（TAMP）相结合，直接根据输入的 RGB 图像和自然语言指令解决多步操作任务。我们的系统旨在简单易用：它可以在不到一小时的时间内安装并在标准的 DROID 设置上运行，并能以极小的代价适配到新的机器人本体。我们在仿真和真实世界的 28 个桌面操作任务中评估了 TiPToP（该系统不需要任何机器人数据），发现其表现与 π_{0.5}-DROID 相当甚至更优，后者是一个在 350 小时特定本体演示数据上进行微调的视觉-语言-动作（VLA）模型。TiPToP 的模块化架构使我们能够在组件级别分析系统的故障模式。我们分析了 173 次实验评估的结果，并确定了改进方向。我们开源了 TiPToP，以进一步推动模块化操作系统以及学习与规划之间更紧密集成的研究。项目网站和代码：https://tiptop-robot.github.io

## Abstract
We present TiPToP, an extensible modular system that combines pretrained vision foundation models with an existing Task and Motion Planner (TAMP) to solve multi-step manipulation tasks directly from input RGB images and natural-language instructions. Our system aims to be simple and easy-to-use: it can be installed and run on a standard DROID setup in under one hour and adapted to new embodiments with minimal effort. We evaluate TiPToP -- which requires zero robot data -- over 28 tabletop manipulation tasks in simulation and the real world and find it matches or outperforms $π_{0.5}\text{-DROID}$, a vision-language-action (VLA) model fine-tuned on 350 hours of embodiment-specific demonstrations. TiPToP's modular architecture enables us to analyze the system's failure modes at the component level. We analyze results from an evaluation of 173 trials and identify directions for improvement. We release TiPToP open-source to further research on modular manipulation systems and tighter integration between learning and planning. Project website and code: https://tiptop-robot.github.io

#### 1. 核心问题与整体含义（研究动机和背景）
长期以来，机器人领域的目标是构建一个“开箱即用”的操作系统：能够部署在任意机器人上，通过自然语言指令处理任意物体，且无需针对特定物体、环境或机器人本体进行微调。
*   **现有技术的局限：** 视觉-语言-动作（VLA）模型（如 $\pi_0$、OpenVLA）虽然能直接输出动作，但依赖海量特定数据，且难以跨机器人泛化。传统的任务与运动规划（TAMP）虽然逻辑严密，但通常需要预先知道精确的物体几何模型，且系统集成复杂。
*   **研究目标：** 提出 TiPToP 系统，通过组合预训练的视觉基础模型与 GPU 加速的 TAMP，实现无需任何机器人训练数据、仅凭像素输入和语言指令即可完成多步复杂任务。

#### 2. 方法论
TiPToP 采用模块化架构，将任务分解为感知、规划和执行三个阶段：
*   **感知模块（Perception）：**
    *   **3D 视觉分支：** 使用 *FoundationStereo* 从双目图像预测深度图，并利用 *M2T2* 模型在点云上生成 6-DoF 抓取位姿。
    *   **语义分支：** 使用 *Gemini 1.5 Pro* (VLM) 进行目标检测和语义目标接地（Goal Grounding），将语言指令转化为逻辑谓词（如 `On(A, B)`）；使用 *SAM-2* 进行像素级分割。
    *   **整合：** 将分割掩码与点云结合，通过凸包（Convex Hull）重建每个物体的 3D 网格模型。
*   **规划模块（Planning）：**
    *   使用 **cuTAMP**（一种 GPU 并行化的 TAMP 算法）。它首先枚举候选的动作骨架（如：移动->抓取->移动->放置），然后利用可微优化同时搜索抓取位姿、放置位置和避障轨迹。
*   **执行模块（Execution）：**
    *   采用自定义的关节阻抗控制器，精确跟踪规划出的时间参数化轨迹。

#### 3. 实验设计
*   **实验场景：** 涵盖仿真环境（IsaacSim）和真实世界（DROID 硬件平台、UR5e、WidowX AI）。
*   **任务分类：** 28 个任务分为四类：简单任务、带干扰物任务、语义推理任务（如“把最大的玩具放进盘子”）、多步任务（如需先移开障碍物再抓取）。
*   **Benchmark 与对比方法：**
    *   主要对比对象是 **$\pi_{0.5}$-DROID**，这是一个在 350 小时特定机器人演示数据上微调过的最先进 VLA 模型。
    *   引入了**外部评估团队**（非开发人员）独立部署并测试系统，以验证易用性和客观性。

#### 4. 资源与算力
*   **训练算力：** TiPToP **无需任何机器人特定数据的训练**，直接使用现成的基础模型。
*   **推理算力：** 实验中使用了多种 GPU，包括 NVIDIA L4（仿真）、RTX 3080 Laptop（外部评估）和 RTX 4090（系统开发方）。
*   **部署效率：** 系统宣称可在 1 小时内完成安装并部署到标准 DROID 设置上。

#### 5. 实验数量与充分性
*   **实验规模：** 进行了 165 次对比实验（涵盖 28 个场景）和 173 次额外的故障分析实验。
*   **充分性：** 实验设计较为充分，不仅有仿真和实验室内部测试，还包含了第三方外部评估，增强了结果的可信度。
*   **客观性：** 通过对比成功率（SR）和任务进度（TP）两个指标，客观展示了规划系统与端到端学习模型在不同维度下的优劣。

#### 6. 主要结论与发现
*   **性能超越：** 在无需任何机器人数据的情况下，TiPToP 在语义推理和多步规划任务上的表现显著优于经过大量微调的 $\pi_{0.5}$-DROID。
*   **执行效率：** TiPToP 生成的轨迹通常比 VLA 模型的反应式控制更直接、速度更快（完成时间缩短约 50%）。
*   **故障模式：** 故障分析显示，抓取失败（31/55）是主要瓶颈，其次是感知导致的几何近似误差。
*   **泛化性：** 成功在 UR5e 和 WidowX 等不同机器人本体上快速部署，证明了其模块化设计的灵活性。

#### 7. 优点
*   **零样本能力：** 彻底摆脱了对机器人操作演示数据的依赖。
*   **模块化可解释性：** 失败可以追溯到具体的感知或规划组件，便于针对性调试和升级。
*   **强大的语义理解：** 借助大语言/视觉模型，能够处理复杂的开放词汇指令。
*   **高效规划：** cuTAMP 的引入解决了传统 TAMP 计算缓慢的问题。

#### 8. 不足与局限
*   **开环执行：** 目前系统在执行阶段不进行视觉反馈（Open-loop），一旦物体发生意外位移或抓取打滑，系统无法实时纠正。
*   **感知局限：** 依赖单视角感知，凸包近似法无法处理凹面物体（如杯子内部或弯曲的香蕉）。
*   **静态环境假设：** 规划基于初始时刻的快照，难以处理动态变化的环境。
*   **避障保守：** 凸包过大可能导致在拥挤场景中找不到可行路径。
